What I Do

Independent consulting and technical delivery under the TerraDigital brand — geospatial systems, data engineering, and open-source business applications.

TerraDigital

TerraDigital is my consulting brand for open-source geospatial and data engineering work. The name reflects the core idea: territorial intelligence expressed through digital systems. Everything I build runs on open-source infrastructure — PostGIS, QGIS, Django, Python, Docker — without vendor lock-in or licence costs for clients.

The portfolio ranges from custom WebGIS platforms and spatial data pipelines to full-stack business applications where the map is not a decoration but the analytical core of the system.

Areas of Work

WebGIS Platforms

End-to-end geospatial web applications — from PostGIS spatial databases and GeoServer feature services to Leaflet front-ends and Django/GeoDjango backends. Designed for real operational use, not demos.

Spatial Data Engineering

Design and implementation of spatial ETL pipelines, PostGIS schemas, automated data ingestion from public sources (Copernicus, IPMA, NASA FIRMS), and change-data-capture workflows for monitoring environments.

Business Applications with a Territorial Core

Django applications where territorial data — cadastre, georeferenced assets, field collection, route management — is the structural spine of the system, not an afterthought added to a generic CRUD application.

Analytical Systems & Statistical Models

Data pipelines and dashboards for pattern detection and anomaly analysis. Applied statistical methods (Z-scores, Poisson regression, Benjamini-Hochberg FDR correction) on structured event data, delivered as interactive Streamlit or Django applications.

Static Sites & Content Infrastructure

Vanilla HTML/CSS/JS sites hosted on GitHub Pages with zero running costs. Git-based CMS (Sveltia) via Cloudflare Workers OAuth, automated build pipelines with GitHub Actions, and GDPR-compliant cookie management.

Infrastructure & Deployment

Docker Compose stacks on Hetzner VPS with Nginx, Gunicorn, PostgreSQL, Let's Encrypt SSL, and automated backups. Self-hosted, auditable, and free of third-party runtime dependencies.

How I Work

Most of my engagements begin with a client who has a spatial problem they can't quite articulate as a technical specification — a cadastre that doesn't match reality, a field inspection process that runs on spreadsheets and phone calls, or an environmental monitoring requirement with no budget for commercial platforms.

My role is to translate that operational reality into a working system: define the data model, choose the right open-source stack, build iteratively, and hand over something the client can actually maintain. I work alone or in small teams, which means every architectural decision is made by the person who will also write the migration script at 11pm.

  • Open-source only
    No ESRI, no proprietary databases, no SaaS lock-in. The stack is always auditable, portable, and free to run.
  • Delivery over architecture
    A working PostGIS + Django application solving a real problem is worth more than an elegant microservices diagram that never ships.
  • Documented and transferable
    Systems are built to be handed over. Documentation, migration scripts, and operational runbooks are part of the deliverable.
  • Minimum viable infrastructure
    A single well-configured Hetzner VPS handles most workloads. Cloud complexity is introduced only when the problem genuinely demands it.

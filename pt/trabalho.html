O Que Faço

Consultoria independente e desenvolvimento técnico sob a marca TerraDigital — sistemas geoespaciais, engenharia de dados e aplicações de negócio open-source.

TerraDigital

A TerraDigital é a minha marca de consultoria para trabalho geoespacial e engenharia de dados open-source. O nome reflecte a ideia central: inteligência territorial expressa através de sistemas digitais. Tudo o que construo corre sobre infraestrutura open-source — PostGIS, QGIS, Django, Python, Docker — sem dependência de fornecedores nem custos de licenciamento para os clientes.

O portefólio inclui plataformas WebGIS personalizadas, pipelines de dados espaciais e aplicações de negócio completas onde o mapa não é uma decoração mas o núcleo analítico do sistema.

Áreas de Trabalho

Plataformas WebGIS

Aplicações web geoespaciais completas — desde bases de dados PostGIS e serviços GeoServer até frontends Leaflet e backends Django/GeoDjango. Desenhadas para uso operacional real, não para demonstrações.

Engenharia de Dados Espaciais

Desenho e implementação de pipelines ETL espaciais, esquemas PostGIS, ingestão automática de dados de fontes públicas (Copernicus, IPMA, NASA FIRMS) e workflows de monitorização por change-data-capture.

Aplicações de Negócio com Núcleo Territorial

Aplicações Django onde os dados territoriais — cadastro, activos georeferenciados, recolha em campo, gestão de rotas — são a espinha dorsal do sistema, não um módulo acrescentado a posteriori.

Sistemas Analíticos e Modelos Estatísticos

Pipelines de dados e dashboards para detecção de padrões e análise de anomalias. Métodos estatísticos aplicados (Z-scores, regressão de Poisson, correcção FDR de Benjamini-Hochberg) sobre dados de eventos estruturados.

Sites Estáticos e Infraestrutura de Conteúdo

Sites em HTML/CSS/JS vanilla alojados no GitHub Pages sem custos de funcionamento. CMS baseado em Git (Sveltia) via Cloudflare Workers OAuth, pipelines de build automatizados com GitHub Actions e gestão de cookies conforme o RGPD.

Infraestrutura e Deployment

Stacks Docker Compose em VPS Hetzner com Nginx, Gunicorn, PostgreSQL, SSL Let's Encrypt e backups automatizados. Auto-alojado, auditável e sem dependências de runtime de terceiros.

Como Trabalho

A maioria dos meus projectos começa com um cliente que tem um problema espacial que não consegue articular como especificação técnica — um cadastro que não corresponde à realidade, um processo de inspecção em campo que corre em folhas de cálculo e chamadas telefónicas, ou um requisito de monitorização ambiental sem orçamento para plataformas comerciais.

O meu papel é traduzir essa realidade operacional num sistema funcional: definir o modelo de dados, escolher a stack open-source adequada, construir de forma iterativa e entregar algo que o cliente consiga efectivamente manter. Trabalho sozinho ou em equipas pequenas, o que significa que cada decisão arquitectural é tomada pela mesma pessoa que também escreve o script de migração às 23h.

  • Exclusivamente open-source
    Sem ESRI, sem bases de dados proprietárias, sem dependência de SaaS. A stack é sempre auditável, portátil e gratuita para correr.
  • Entrega antes de arquitectura
    Uma aplicação PostGIS + Django a funcionar e a resolver um problema real vale mais do que um diagrama elegante de microsserviços que nunca chega a produção.
  • Documentado e transferível
    Os sistemas são construídos para ser entregues. Documentação, scripts de migração e runbooks operacionais fazem parte do entregável.
  • Infraestrutura mínima viável
    Um único VPS Hetzner bem configurado suporta a maioria das cargas de trabalho. A complexidade cloud só é introduzida quando o problema genuinamente o exige.
